# 📝 AI Prompt 提示词指南

自定义 AI 提示词，让 AI 按你的要求生成书签描述。

## 🚀 快速开始

1. 登录系统，进入「设置」→「AI 助手」
2. 找到「自定义 Prompt 提示词」区域
3. 开启「启用自定义 Prompt」开关
4. 在文本框中输入你的提示词，或点击「填充示例模板」快速填充
5. 点击「保存 Prompt」

## 📝 可用变量

在提示词中可以使用以下变量，系统会自动替换：

### 书签描述生成

- `{name}` - 书签名称
- `{url}` - 书签链接

### 分类推荐

如果提示词中包含 `{categories}` 变量，则会用于**分类推荐**功能而不是书签描述生成：

- `{name}` - 书签名称
- `{url}` - 书签链接
- `{description}` - 书签描述（可能为空）
- `{categories}` - 现有分类列表（格式：ID: 分类路径）

> **注意**：分类推荐和书签描述是两个独立的功能。如果自定义提示词中包含 `{categories}` 变量，将只用于分类推荐；否则用于书签描述生成。

## 💡 示例模板

### 书签描述生成示例

#### 示例 1：简洁中文（推荐）

```
为以下书签生成简洁的中文描述：

名称：{name}
链接：{url}

要求：
1. 使用简体中文
2. 一句话说明网站功能，20字以内
3. 直接返回描述，不要引号

示例：
- GitHub → 全球最大的代码托管平台
- 知乎 → 中文互联网问答社区
```

#### 示例 2：详细描述

```
请为以下网站生成详细的中文介绍：

网站名称：{name}
网站地址：{url}

要求：
1. 简体中文
2. 2-3句话，80字左右
3. 说明网站性质、核心功能、适用人群
4. 纯文本输出

示例：
GitHub 是全球最大的代码托管和协作开发平台，提供 Git 版本控制、项目管理、代码审查等功能。适合开发者进行开源项目开发、团队协作和技术学习。
```

#### 示例 3：极简风格

```
网站：{name}
链接：{url}

用一句话（15字内）概括这个网站。
使用简体中文，直接输出。
```

### 分类推荐示例

#### 示例 1：智能分类（推荐）

```
你是一个书签分类助手，需要为书签选择最合适的分类。

书签信息：
- 名称：{name}
- 链接：{url}
- 描述：{description}

现有分类列表：
{categories}

要求：
1. 从上述分类列表中选择最合适的一个分类 ID
2. 分析书签的主题、功能、用途
3. 返回 JSON 格式：{"categoryId": 选中的ID, "reason": "选择理由（简体中文，一句话）"}

示例：
- GitHub（代码托管） → 开发工具分类，理由：开发者常用的代码管理工具
- 知乎（问答社区） → 社交媒体分类，理由：中文知识问答与讨论平台
```

#### 示例 2：精准匹配

```
根据书签内容选择最匹配的分类。

书签：{name}
地址：{url}
说明：{description}

可选分类（ID: 路径）：
{categories}

返回 JSON 格式：
{"categoryId": 数字ID, "reason": "中文说明"}

选择标准：
1. 优先匹配功能类型（如开发工具、设计资源）
2. 其次匹配主题领域（如技术、生活）
3. 最后选择通用分类（如其他、收藏）
```

#### 示例 3：简洁风格

```
为书签 "{name}" ({url}) 选择分类。

描述：{description}

分类列表：
{categories}

返回格式：{"categoryId": ID, "reason": "理由"}
要求：使用简体中文，理由控制在20字内。
```

## ⚙️ 使用技巧

### 书签描述生成

1. **控制长度**：在提示词中明确指定字数（如"20字以内"）
2. **指定语言**：要求"使用简体中文"确保输出语言
3. **提供示例**：给出 2-3 个示例帮助 AI 理解格式
4. **保持简洁**：提示词不要超过 300 字
5. **快速开始**：点击「填充示例模板」按钮可以快速填充预设模板，然后根据需要修改

### 分类推荐

1. **必须包含 {categories} 变量**：这是识别分类推荐功能的关键
2. **明确 JSON 格式**：要求返回 `{"categoryId": 数字, "reason": "理由"}` 格式
3. **说明选择标准**：告诉 AI 如何判断最合适的分类（功能、主题、领域等）
4. **使用中文理由**：要求 reason 字段使用简体中文，便于用户理解
5. **提供分类示例**：给出几个书签→分类的配对示例

## 🔄 恢复默认

想要恢复使用系统默认提示词？只需关闭「启用自定义 Prompt」开关并保存即可。

## ❓ 常见问题

**Q：如何启用自定义 Prompt？**  
A：进入设置页面，开启「启用自定义 Prompt」开关，填写或使用示例模板，然后保存即可。

**Q：AI 生成的内容不是中文？**  
A：在提示词中明确指定"使用简体中文"，并提供中文示例。

**Q：生成的描述太长或太短？**  
A：在提示词中加入明确的字数限制，如"20字以内"或"50-80字"。

**Q：能否为分类推荐功能自定义提示词？**  
A：可以。分类推荐有独立的默认提示词。如需自定义，请在提示词中包含 `{categories}` 变量（必需），系统会自动识别并将其用于分类推荐功能。详见上方"分类推荐示例"章节。

**Q：分类推荐和书签描述有什么区别？**  
A：书签描述用于生成网站简介文字，分类推荐用于自动选择合适的分类。如果自定义提示词包含 `{categories}` 变量，则只用于分类推荐；否则用于书签描述生成。

**Q：分类推荐返回格式有什么要求？**  
A：必须返回 JSON 格式：`{"categoryId": 分类ID数字, "reason": "选择理由"}`。确保在提示词中明确说明这个格式要求。
